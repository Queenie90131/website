模板 A · 深色赛博朋克风项目汇报
【角色设定】 你是一位资深前端设计师和数据可视化专家，擅长制作科技感十足的 HTML 汇报文档。 【风格定义】 - 整体风格：深色赛博朋克科技感，主色调为深蓝黑（#0a1628） - 强调色：霓虹蓝（#00b4d8）+ 橙色（#ff8c00） - 卡片使用毛玻璃效果（backdrop-filter: blur） - 背景包含低透明度网格线和粒子飘浮效果 - 字体使用系统无衬线字体，数据数字使用等宽字体 【内容结构】 请生成一个完整的单页 HTML 文档，包含以下模块： 1. 封面区域：项目标题 + 副标题 + 关键数据概览（3-4 个数字指标） 2. 项目路线图：时间轴形式展示 [填写你的项目阶段]，每个节点包含日期和里程碑 3. 系统架构图：分层展示 [填写你的架构层级]，使用卡片堆叠效果 4. 数据图表区： - 柱状图展示 [填写对比数据] - 环形图展示 [填写占比数据] - 趋势折线图展示 [填写趋势数据] 5. 关键成果卡片：4-6 张卡片展示核心成果 【动效要求】 - 页面加载时各模块依次渐入（stagger 0.1s） - 数字指标从 0 滚动到目标值（duration 1.5s） - 卡片 hover 时轻微上浮（transform: translateY(-2px)） - 路线图节点依次点亮 - 图表数据条从 0 填充到目标值 - 背景粒子缓慢飘浮 - 所有动效使用 ease-in-out 缓动，克制不花哨 【技术约束】 - 纯 HTML 单文件，CSS 和 JS 全部内联 - 图表使用纯 CSS 实现或引入 Chart.js CDN - 响应式设计，适配桌面端（1200px）和平板端 - 不使用任何需要安装的依赖 - 确保在 Chrome / Safari / Firefox 中正常显示
模板 B · 简约科技白产品战略汇报
【角色设定】 你是一位 Apple/Stripe 风格的 UI 设计师，擅长制作极简而精致的 HTML 汇报文档。 【风格定义】 - 整体风格：简约科技白，大量留白，高对比度 - 主背景：近白色（#f8f9fc），卡片纯白（#ffffff） - 强调色：深蓝（#1a1a6c）+ 亮蓝（#4361ee） - 圆角卡片 + 微阴影（box-shadow: 0 2px 12px rgba(0,0,0,0.06)） - 字体层次分明：标题 28px/700，正文 15px/400，辅助 13px/500 【内容结构】 请生成一个完整的单页 HTML 文档，包含以下模块： 1. 战略概览：一句话愿景 + 3 个核心战略方向卡片 2. 产品路线图：横向时间轴，展示 [填写产品阶段] 3. 市场数据： - 大数字 KPI 展示（TAM/SAM/SOM） - 市场份额饼图 - 增长趋势面积图 4. 竞争格局：2x2 矩阵或对比表格 5. 行动计划：优先级矩阵 + 下一步行动清单 【动效要求】 - 滚动时各区块从下方优雅渐入（opacity 0→1, translateY 20px→0） - KPI 数字计数动画（duration 2s, ease-out） - 卡片 hover 时阴影加深 + 轻微上浮 - 图表数据平滑绘制动画 - 保持克制：同一页面同时出现的动效不超过 3 个 【技术约束】 - 纯 HTML 单文件，内联 CSS + JS - 图表使用 Chart.js CDN（轻量级） - 支持 prefers-reduced-motion 媒体查询 - 响应式，移动端友好 - 所有资源使用 CDN，无需本地依赖
模板 C · 数据仪表盘风业务总结
【角色设定】 你是一位数据可视化专家，擅长制作 Grafana/数据大屏风格的 HTML 业务仪表盘。 【风格定义】 - 整体风格：数据仪表盘，模块化网格布局 - 背景：深灰蓝（#0d1117） - 卡片：深色半透明（rgba(22, 27, 34, 0.8)）+ 细边框 - 强调色：绿（#3fb950）+ 蓝（#58a6ff）+ 橙（#d29922）+ 红（#f85149） - 数据数字使用等宽字体，颜色根据正负变化 【内容结构】 请生成一个完整的单页 HTML 文档，包含以下模块： 1. 顶部 KPI 栏：4-6 个关键指标卡片（带同比/环比变化箭头） 2. 核心图表区（2x2 网格）： - 收入趋势折线图（近 12 个月） - 业务占比环形图 - 区域分布柱状图 - 转化率漏斗图 3. 排行榜：Top 5 产品/区域/团队的横向条形图 4. 异常预警区：高亮显示需要关注的指标 【动效要求】 - 数据加载时数值从 0 滚动到目标值 - 图表依次绘制（stagger 0.2s） - 异常指标卡片有轻微红色呼吸效果 - 实时时钟显示（右上角） - 数据刷新按钮 + 加载动画 【技术约束】 - 纯 HTML 单文件，内联 CSS + JS - 图表使用 Chart.js CDN - 使用 CSS Grid 实现仪表盘布局 - 支持窗口 resize 自适应 - 模拟数据即可，无需真实 API 连接

提示词优化技巧

✓
给具体数值而非模糊描述
好："卡片圆角 12px，间距 20px，字体 14px"
差："卡片好看一点，间距适中"
✓
用参考锚定风格
好："参考 Stripe 官网的视觉风格"
差："要高级感"
✓
分步迭代而非一步到位
好：先出结构 → 再调样式 → 最后加动效
差：一次性要求所有细节完美
✓
明确「不要什么」
好："不要弹跳动画，不要彩虹色"
差：只说"要克制"

汇报内容组织方法论

六大经典框架，从结构化思维到故事化表达，让你的汇报逻辑清晰、说服有力。

框架 核心思想 最佳场景 难度
金字塔原理 结论先行，以上统下 所有正式汇报 入门
SCQA 模型 情境→冲突→问题→答案 项目/问题驱动型汇报 入门
STAR 法则 情境→任务→行动→结果 绩效/案例汇报 入门
AIDA 模型 注意→兴趣→欲望→行动 产品发布/战略说服 中级
4P 叙事法 图片→承诺→证明→推动 商业提案/融资路演 中级
黄金圈法则 Why→How→What 愿景型/团队激励 进阶
金字塔原理
Minto Pyramid Principle · 芭芭拉·明托
结论先行 → 以上统下 → 归类分组 → 逻辑递进
核心结论 支撑论点 A 支撑论点 B 支撑论点 C 论据/数据
适用场景：向高管汇报、项目评审、任何需要快速传达核心观点的场合。
一句话口诀："先说答案，再说理由，最后给证据。"
SCQA 模型
Situation - Complication - Question - Answer
情境(S) → 冲突(C) → 问题(Q) → 答案(A)
背景现状 遇到的问题 核心疑问 解决方案
适用场景：项目复盘、问题驱动型汇报、需要引起共鸣的开场。
一句话口诀："本来挺好，但是出了事，怎么办？我的方案是..."
STAR 法则
Situation - Task - Action - Result
情境(S) → 任务(T) → 行动(A) → 结果(R)
背景 目标 做了什么 取得了什么
适用场景：个人/团队绩效汇报、案例分享、面试述职。
一句话口诀："当时什么情况，我要做什么，我做了什么，结果怎样。"
AIDA 模型
Attention - Interest - Desire - Action
注意(A) → 兴趣(I) → 欲望(D) → 行动(A)
抓住眼球 引发兴趣 激发渴望 促成行动
适用场景：产品发布、战略说服、需要推动决策的汇报。
一句话口诀："你看这个！→ 是不是很有意思？→ 你也需要！→ 现在就做！"
4P 叙事法
Picture - Promise - Proof - Push
图片(P) → 承诺(P) → 证明(P) → 推动(P)
描绘愿景 做出承诺 给出证据 推动决策
适用场景：商业提案、融资路演、需要获取资源支持的汇报。
一句话口诀："想象一下...→ 我保证能...→ 证据在这里...→ 请批准/投资。"
黄金圈法则
Golden Circle · Simon Sinek · Why-How-What
Why（为什么）→ How（怎么做）→ What（做什么）
信念/使命 方法/路径 产品/成果
适用场景：愿景型汇报、团队激励、品牌故事、年度战略发布。
一句话口诀："我们相信...→ 所以我们...→ 最终做出了..."

框架选择决策树

不确定用哪个框架？按以下路径快速判断：

你的汇报目的是？
传达结论/决策 → 金字塔原理
讲述一个项目/案例 → STAR 法则
提出并解决问题 → SCQA 模型
说服对方采取行动 → AIDA 模型
获取资源/投资 → 4P 叙事法
传递愿景/激励团队 → 黄金圈法则

动效设计原则

克制且有效的微动效设计——让动效服务于信息，而非炫技。

五大核心原则

1
服务于信息
动效引导视线、强调重点，不喧宾夺主
2
时间控制
微动效 ≤ 300ms，过渡动画 ≤ 800ms
3
缓动函数
ease-in-out 为主，避免线性或过度弹跳
4
一致性
统一动效语言，同类元素用同类动效
5
可访问性
支持 prefers-reduced-motion 媒体查询
