📄 模块一：富文本内容同步（飞书文档 → Obsidian）
1

安装 Lark Sync 插件（推荐，开箱即用）

在 Obsidian 中打开 设置 → 社区插件 → 浏览，搜索 "Lark Sync" 并安装。
🔗 GitHub 地址 | 开发者：zhuxiaoer（中文）

2

配置飞书开放平台（获取 API 凭证）

访问 飞书开放平台 → 创建「企业自建应用」→ 获取 App ID 和 App Secret。
在「权限管理」中添加：文档：docs:document:readonly 云空间：drive:drive:readonly

3

在 Obsidian 中配置 Lark Sync

填入 App ID / App Secret，选择要同步的飞书知识库文件夹，设置本地存储路径。
支持手动同步 / 定时自动同步 / Webhook 实时同步。

💡 替代方案：如果你偏好自部署，可以用 feishu2md（支持 Docker 部署），命令示例：
feishu2md --app-id xxx --app-secret xxx --wiki-link output ./obsidian-vault
📊 模块二：多维表格（飞书多维表格 → Obsidian DataView）
1

安装 DataView 插件

Obsidian → 设置 → 社区插件 → 浏览 → 搜索 "DataView" 安装。

导出飞书多维表格

打开飞书多维表格 → 右上角菜单 → 导出为 CSV 或通过 API 获取 JSON。

使用 Templater 插件（🔗 下载）创建模板，自动将 CSV 每行生成为单篇笔记，frontmatter 保留字段。示例如下：

---

优先级: P0
负责人: 张三

用 DataView 重建表格视图

在笔记中添加查询块，展示可排序、筛选的动态表格：

FROM ""
WHERE type = "产品需求"
SORT 优先级 DESC

📄 模块五：本地 HTML 资料如何进入 Obsidian 知识库

方案对比总览

方案方法保留程度劳动量推荐场景
① 自动转换 HTML → Markdown用工具批量转为 .md 文件⭐⭐⭐低首选 标准 HTML 文档
② 嵌入 HTML 到 Obsidian用 iframe/embed 直接引用⭐⭐⭐⭐⭐极低含复杂交互/JS 的 HTML
③ 存为附件 + 链接索引HTML 原文件放附件，笔记做索引⭐⭐⭐⭐⭐低需要完整保真
④ 截图 + OCR 全文检索截图保存，OCR 提取文字⭐⭐中临时性/无法解析的格式
⑤ 手动复制粘贴重写人工逐篇复制到新笔记⭐⭐⭐⭐极高少量精华内容

方案详解

1

方案 ①：自动转换 HTML → Markdown 推荐首选

使用 Pandoc 或 Node.js 工具批量将 .html 转为 .md，直接放入 Obsidian 仓库。

工具 A：Pandoc（最强大，支持批量）
🔗 下载 Pandoc（支持 macOS / Win / Linux）
macOS 安装 brew install pandoc
批量转换命令
for f in *.html; do pandoc "$f" -f html -t markdown -o "${f%.html}.md"; done

工具 B：html-to-md（Node.js，适合中文）
🔗 GitHub 地址
安装 npm install -g html-to-md
转换 html-to-md input.html -o output.md

工具 C：Obsidian 插件
- 社区搜索 HTML Reader：在 Obsidian 中直接读取 .html 文件
- 社区搜索 Import HTML：一键将 HTML 导入为 Markdown 笔记
✅ 推荐工作流
1. 用 Pandoc 批量转换所有 HTML → Markdown
2. 转换后放入 Obsidian 仓库的 /docs/ 目录
3. 用 Omnisearch 做全文检索
4. 用 DataView 为文档补充 frontmatter 元数据
2

方案 ②：在 Obsidian 中嵌入 HTML（保真方案）

不转换，直接把 HTML 放在 Obsidian 仓库中，通过嵌入方式引用。

方法 A：HTML embed 标签
在笔记中写：
<embed src="attachments/report.html" width="100%" height="500px">
注意 需在 Obsidian 设置中开启「允许 HTML 渲染」

方法 B：Obsidian iframe 插件
安装社区插件 Obsidian iframe 或 Obsidian Webview

方法 C：HTML Reader 插件
双击 .html 文件即可在 Obsidian 中显示渲染页面
3

方案 ③：附件存储 + 索引笔记

HTML 原文件放在 Obsidian 附件目录，用笔记做索引目录。

目录结构建议
obsidian-vault/
├── attachments/
│   ├── html-archive/
│   │   ├── report-2023.html
│   │   ├── spec-v2.html
│   │   └── ...
│   └── images/
├── 📌 HTML 资料索引.md
└── 📂 按主题分类_产品设计.md
✅ 推荐工作流
1. HTML 文件存入 attachments/html-archive/
2. 用 Pandoc 提取标题/摘要写入 frontmatter
3. 创建索引笔记，用 DataView 展示一览
4. 安装 HTML Reader 插件，双击即可预览
4

方案 ④：截图 + OCR 全文检索

浏览器打开 HTML → 截图 → OCR 提取文字。适用临时性或无法解析的格式。

工具推荐
- macOS 自带：预览 App + Cmd+Shift+4 截图
- Text Extractor 插件：社区搜索安装，自动提取图片文字
- Snipaste：国产截图工具
- Omnisearch + Text Extractor：搜索图片中的文字
5

方案 ⑤：人工阅读 → 重写成笔记

最费时但最有内化效果。适合精华内容。

推荐配合插件
- Markdown Table Editor：方便编辑表格
- Templater：创建笔记模板
- Sidebar TOC：自动生成目录
🌐 模块六：本地 HTML 在线托管方案（低成本 · 长期稳定）

如果你有 HTML 文档需要在线上打开查看，以下是四种低成本的托管方案。

各方案详细步骤

1

GitHub Pages + Cloudflare

1. 注册 GitHub → New repository
2. 仓库名 <用户名>.github.io → Public → Create
3. Add file → Upload files → 上传 HTML
4. Settings → Pages → Source: Deploy from branch → main → Save
5. 访问 https://<用户名>.github.io/<文件>.html

加速中国：注册 Cloudflare（免费）→ 添加域名 → 自动 CDN
2

Cloudflare Pages（中国访问最快的免费方案）

1. 注册 Cloudflare（免费）
2. Workers & Pages → Pages → Connect to Git
3. 授权 GitHub，选择 HTML 仓库
4. Project name 填任意名 → Framework: None → Output dir: /
5. Save and Deploy → 访问 https://<项目>.pages.dev/<文件>.html

更新方式：推送新文件到 GitHub，Cloudflare 自动部署
3

Vercel（最快速部署）

1. 注册 Vercel（GitHub 登录）
2. Add New → Project → 选择 HTML 仓库
3. Framework: Other → Deploy
4. 访问 https://<项目>.vercel.app/<文件>.html

注意 中国境内无节点，访问速度一般
4

阿里云 OSS + 静态网站托管（中国大陆最佳速度）

1. 注册 阿里云 → 搜索「对象存储 OSS」
2. 创建 Bucket：唯一名称 → 地域选最近 → 权限「公共读」
3. 基础设置 → 静态页面 → 开启
4. 文件管理 → 上传文件 → 拖入 HTML
5. 访问 https://<bucket>.oss-cn-hangzhou.aliyuncs.com/<文件>.html

费用 存储 0.12 元/GB/月 + 流量 0.5 元/GB ≈ 1-9 元/月
🎯 推荐策略
先走 Cloudflare Pages — 完全免费、亚太节点多、中国速度快。
如果速度不理想 → 切换 阿里云 OSS，每月几块钱。

🔌
网络问题：如果社区插件无法加载，设置中开启「受限模式」并配置国内镜像源，或手动下载插件 .zip 放入 .obsidian/plugins/
