01｜一句话总览：你要搭建的不是笔记软件，而是“会被 AI 维护的个人知识库”

传统第二大脑的核心是“我把资料整理好，未来我能找到”。Karpathy 的 LLM Wiki 思想把角色倒过来：人负责选择高质量信息与提出问题，AI 负责把信息编译成可复用、可链接、可演化的 Wiki。

→
AI Ingest阅读、抽取、归纳
→
LLM Wiki概念页、项目页、人物页
→
Obsidian链接、图谱、Canvas
→
Output决策 / 汇报 / 产品 / 内容
1

知识编译

不是把资料堆进仓库，而是把原始信息加工为结构化 Wiki 页面。

2

AI 维护

AI 持续做摘要、去重、补链接、发现冲突、更新索引，人不用手工维护所有细节。

3

能力复利

每一次阅读、会议、研究、复盘都沉淀到知识库，未来可被再次调用。

最终目标：当你问“我该怎么做巴西外卖市场进入策略？”时，AI 不是临时上网乱搜，而是先读取你的行业研究、面试资料、产品经验、商业判断框架，再生成更贴合你的答案。

02｜理论基础：Karpathy LLM Wiki + PARA + CODE

A. Karpathy 的 LLM Wiki：从 Retrieve 到 Compile

范式传统 RAG / 搜索式知识库LLM Wiki / 编译式知识库
核心动作每次提问时临时检索原始资料先把资料整理成长期 Wiki，提问时读取 Wiki
知识状态碎片化、上下文短、容易重复理解结构化、可链接、可持续更新

B. PARA：给非程序员最容易坚持的目录骨架

PARA 是 Tiago Forte 提出的信息组织法：Projects（项目）、Areas（领域）、Resources（资源）、Archives（归档）。它的好处是简单、行动导向，不需要你一开始设计过度复杂的分类体系。

Projects

项目

有明确目标和截止时间。例如：Keeta 面试准备、AI SaaS MVP、巴西市场研究。

Areas

领域

长期需要维护的责任。例如：产品能力、海外业务、个人品牌、投资学习。

Resources

资源

感兴趣但暂时不直接行动的主题。例如：AI Agent、Obsidian、外卖平台案例。

Archives

归档

已经完成、暂停或暂时不用的信息，避免污染当前工作区。

03｜Obsidian 操作指南：你需要掌握的 20% 功能

Obsidian 的本质是一个本地 Markdown 知识库。它强大的地方不是“写字”，而是把每一条知识变成可链接、可图谱化、可被 AI 读取的文件。

1. 必懂术语解释

术语中文解释为什么重要
Vault库 / 知识仓库，一个本地文件夹你的所有笔记、附件、模板都放在这里，AI 也读取这个文件夹。
Markdown一种纯文本写作格式不依赖某个软件，方便迁移、版本管理和 AI 处理。
Wikilink[[双向链接]]把笔记连成网络，例如 [[巴西外卖市场]] 连接到 [[Keeta]]。
Backlink反向链接看到哪些笔记提到当前概念，帮助发现隐藏关系。
Graph View知识图谱视图用图展示笔记之间的关系，适合观察知识网络。
Canvas无限白板适合做框架图、研究地图、产品策略图。
Properties / Frontmatter笔记顶部的结构化字段给 AI 和你自己提供元数据，如状态、来源、标签、更新时间。

3. 第一次安装与配置

下载 Obsidian
到 Obsidian 官网下载安装 Mac 版。打开后选择 Create new vault，命名为 AI-Second-Brain。
打开核心功能
Settings → Core plugins：开启 Backlinks、Graph view、Canvas、Templates、Daily notes、Command palette。
设置链接规则
Settings → Files and links：建议开启 Automatically update internal links，并使用 Wikilinks。这样重命名笔记时链接会自动更新。

创建 90_Templates，并在 Templates 插件中指定该文件夹。后续用模板快速创建项目页、概念页、资料卡。

# Obsidian Ingest Skill

## 目标
把输入资料转化为可长期复用的 Wiki 知识，而不是简单摘要。

## 工作流程
1. 判断资料类型：文章 / PDF / 会议 / 视频 / 想法。
2. 提取核心观点、事实、人物、公司、概念。
3. 更新或创建对应的 Wiki 页面。
4. 添加双向链接，例如 [[AI Agent]]、[[巴西外卖市场]]。
5. 在 Sources 中记录来源。
6. 输出本次变更摘要。

## 输出原则
- 不确定的内容必须标注 Unknown 或 Need verification。
- 不要删除原始资料。
- 优先更新已有页面，避免重复页面。

2. 你需要的 6 个核心 Skill

Ingest

资料摄取 Skill

把网页、PDF、会议纪要、视频笔记转成 Obsidian 结构化笔记。

Wiki

Wiki 编译 Skill

把零散笔记合并为概念页、人物页、公司页、方法论页。

Research

研究 Skill

围绕一个问题做深度研究，并把结论沉淀回 Vault。

Lint

知识健康检查 Skill

查重复、孤立页面、冲突观点、缺少来源、过期内容。

Output

输出 Skill

把知识库内容转化为汇报、文章、面试稿、产品方案。

Review

复盘 Skill

每周总结新增知识、关键洞察、下一步行动。

安全提醒：第三方 Skill 本质上是会影响 Agent 行为的说明文件，甚至可能包含危险指令。不要随意运行来路不明的 Skill；使用前先打开 SKILL.md 阅读，确认它不会要求上传隐私、删除文件、运行不明脚本或泄露 API Key。

创建 Vault

# AI Second Brain Operating Rules

你正在维护一个 Obsidian Vault。目标是把用户的资料编译成长期可复用的个人 Wiki。

## 总原则
- 不要只做摘要；要把信息沉淀为可复用知识。
- 优先更新已有页面，避免重复创建相似页面。
- 所有新概念尽量使用 [[Wikilink]]。
- 不确定的信息必须标注 confidence: low 或 needs_verification。
- 不要删除原始资料。

## 目录规则
- 00_Inbox：未经处理的原始输入
- 01_Projects：有目标和截止时间的项目
- 02_Areas：长期能力和责任
- 03_Resources：主题资源
- 04_Wiki：AI 编译后的概念页
- 05_Outputs：最终输出
- 06_Reviews：复盘
- 99_Archives：归档

## 每次处理资料时必须输出
1. 创建/更新了哪些页面
2. 发现了哪些新概念
3. 建立了哪些链接
4. 哪些内容需要进一步验证

建议第一篇资料就选 Karpathy LLM Wiki 或一篇你正在学习的 AI Agent 文章。你要观察 AI 是否能输出：

  • ✅ 一篇 Source Note：保留原始来源、作者、日期、摘要。
  • ✅ 多篇 Wiki Note：例如 LLM Wiki.md、RAG.md、Agent Skill.md。
  • ✅ 双向链接：在笔记中出现 [[Obsidian]]、[[Knowledge Graph]]。
  • ✅ 变更摘要：告诉你它新增和更新了什么。

工具路线适合谁怎么用难度

06｜四条核心工作流：让第二大脑真正服务你的创业与学习

工作流 1：每日输入

目标：每天把外部高价值信息进入 Inbox。

看到好文章 → 复制链接/摘要 → 放入 00_Inbox → 标记 source_type

最低标准：每天 1 条即可，宁少勿滥。

工作流 2：AI 编译

目标：把输入转成 Wiki。

Inbox 资料 → AI 提炼 → 更新 04_Wiki → 建立 [[链接]] → 标记来源

最低标准：每周处理 3 条。

工作流 3：项目输出

目标：把知识用于真实成果。

选择项目 → 检索相关 Wiki → 生成框架 → 输出方案/汇报/文章

最低标准：每周 1 个可分享产出。

工作流 4：知识 Lint

目标：保持知识库健康。

查重复 → 查孤立页 → 查冲突 → 查缺来源 → 生成修复建议

最低标准：每周一次。
